## Session Objective
Build "Muesli" — a local-first macOS menu bar app combining WisprFlow (push-to-talk dictation) and Granola (meeting transcription with speaker diarization) functionality. Zero cloud STT costs via on-device mlx-whisper on Apple Silicon. Target: M4 MacBook Air 16GB with ~3.8GB reclaimable RAM under typical load.

## What Got Done

### Phase 1: Dictation MVP (COMPLETE + manually tested)
- `audio/mic_capture.py` — Always-open `sounddevice.InputStream` with queue-based callback. Stream runs continuously; audio discarded when not recording (privacy safe). 16kHz mono float32.
- `transcribe/engine.py` — mlx-whisper wrapper. Lazy-loads `mlx-community/whisper-small.en-mlx` (~500MB). Auto-unloads after 120s idle. `preload()` called at startup in background thread. Model loads in ~2.1s.
- `dictation/hotkey.py` — Left Cmd hold-to-record. 0.15s timer distinguishes hold (dictation) from Cmd+C shortcuts. Cancels if any other key pressed while Cmd held.
- `dictation/paste.py` — `pyperclip.copy()` + pynput `Cmd+V` simulation to paste into active app.
- `app.py` — rumps menu bar app. Microphone emoji icon. Dictation + meeting controls + recent meetings submenu.
- `main.py` — Entry point.
- `config.py` — JSON config at `~/Library/Application Support/Muesli/config.json`.

### Phase 2: Meeting Recording (COMPLETE but NOT manually tested)
- `audio/system_capture.py` — ScreenCaptureKit via pyobjc. `_AudioDelegate` NSObject receives audio samples. Video set to 1x1px@1fps to minimize overhead. `setExcludesCurrentProcessAudio_(True)`. Requires Screen Recording permission (macOS prompts on first use).
- `cal_monitor/monitor.py` — EventKit polling every 60s. Detects meetings within 5min with Zoom/Meet/Teams/Webex URLs. Uses `requestFullAccessToEventsWithCompletion_()`.
- `transcribe/batch.py` — Batch WAV transcription in 30s chunks. Returns `list[dict]` with `{"start", "end", "text"}`.
- `meeting/session.py` — Orchestrates mic + system audio, transcription, summary, SQLite save.
- `meeting/transcript.py` — Merges mic ("You") + system ("Others") segments chronologically. Format: `[HH:MM:SS] Speaker: text`.
- `meeting/summary.py` — OpenRouter API (`google/gemma-3-4b-it:free` default). Structured prompt for Summary/Decisions/Action Items/Quotes. Falls back to raw transcript if no API key or error.
- `storage/local_db.py` — SQLite at `~/Library/Application Support/Muesli/muesli.db`. Tables: `meetings`, `dictations`.

## What Didn't Work
- **`calendar/` package shadowed stdlib**: `from calendar import timegm` resolved to our package. → Renamed to `cal_monitor/`.
- **Swift audiograb/audiotee compilation**: SPM builds failed without full Xcode (only CLI tools installed). SwiftBridging module map bug. Direct `swiftc` also failed. → Pivoted to ScreenCaptureKit via PyObjC.
- **rumps double Quit**: `rumps.App(quit_button="Quit")` auto-adds Quit, plus manual `@rumps.clicked("Quit")` added another. → Set `quit_button=None`.
- **rumps.notification() crash**: Missing `Info.plist`/`CFBundleIdentifier`. → Removed notifications entirely (user explicitly didn't want them).
- **Model warm-up hanging**: `import mlx_whisper` doesn't load weights. Must call `load_model(MODEL_REPO)` explicitly to preload. First `transcribe()` was slow without this.
- **Tab+` hotkey**: Inputs characters into text fields. → Switched to Left Cmd hold.
- **Stdout buffering**: Print statements invisible in background tasks. → Added `sys.stdout.flush()` via `_log()` helper.

## Key Decisions
- **Decision**: Single `whisper-small.en` model for both dictation and meetings (not separate models)
  - **Rationale**: M4 Air 16GB has ~3.8GB reclaimable under typical load. Two models = 1GB+. Single model with lazy-load keeps idle at ~45MB, active at ~545MB.

- **Decision**: ScreenCaptureKit for system audio (not audiograb/BlackHole)
  - **Context**: Swift compilation failed without Xcode. BlackHole was installed but adds routing complexity.
  - **Rationale**: ScreenCaptureKit is exactly how Granola captures system audio. Permission-based, no virtual audio device needed. Confirmed by `NSScreenCaptureUsageDescription` in Granola's Info.plist.

- **Decision**: Left Cmd hold-to-record (0.15s delay)
  - **Rationale**: Tab+` types characters. Fn key not available via pynput on macOS. 0.15s timer distinguishes hold (dictation) from Cmd+shortcut combos.

- **Decision**: OpenRouter (not direct API) for meeting summaries
  - **Rationale**: User has OpenRouter account. Free model tier available. Model selection deferred ("decide after core features done").

- **Decision**: Python + rumps, NOT Electron
  - **Rationale**: WisprFlow (142MB Electron) and Granola are both Electron. Muesli at ~45MB idle is lighter. User explicitly asked for non-Electron.

## Lessons Learned
- `mlx_whisper.transcribe()` needs `path_or_hf_repo=MODEL_REPO` passed every call — it's not cached from `load_model()`.
- `rumps.App` `quit_button` param auto-adds a Quit menu item. Set to `None` if adding your own.
- pyobjc ScreenCaptureKit requires async completion handlers via `threading.Event` for synchronization — all SCK operations are async.
- pynput on macOS: `keyboard.Key.cmd` and `keyboard.Key.cmd_l` both fire for left Cmd. Check both.
- MicCapture always-open approach: stream runs continuously, callback only appends when `_recording=True`. No privacy risk (audio is discarded when not recording, never stored).

## Nuances & Edge Cases
- **macOS permissions needed**: Microphone (sounddevice), Accessibility (pynput hotkey+paste), Screen Recording (ScreenCaptureKit system audio), Calendar (EventKit). Each triggers a macOS prompt on first use.
- **No Xcode installed**: Only Command Line Tools. Swift compilation fails. Any future Swift code needs Xcode or pre-built binaries.
- **ScreenCaptureKit video overhead**: Even with audio-only, SCK requires a display filter. Video is minimized to 1x1px@1fps but can't be fully disabled.
- **System audio `_AudioDelegate`** (`system_capture.py:10`): Uses `objc.super()` for proper NSObject initialization. The `stream_didOutputSampleBuffer_ofType_` selector name must match exactly — PyObjC dispatches by selector string.
- **Meeting session creates its own MicCapture** (`session.py:23`): This is separate from the app-level MicCapture used for dictation. The app disables dictation during meeting recording (`app.py:61-62`).
- **Model unload race**: `_try_unload()` checks `time.time() - _last_used >= IDLE_TIMEOUT` under lock to avoid unloading during active transcription.
